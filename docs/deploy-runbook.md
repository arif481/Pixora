# Deployment Runbook (Do This Exactly)

## 0) Pre-check

- Confirm local env files exist:
  - `apps/web/.env.local`
- Generate exact deploy commands from current local values:
  - `bash scripts/deploy_env_prep.sh`

## 1) Deploy web app (Vercel)

1. Install/verify CLI:
   - `npm i -g vercel`
2. Login and link project from repo root:
   - `vercel login`
   - `vercel link`
3. Add production env vars using commands generated by `scripts/deploy_env_prep.sh`.
4. Deploy:
   - `vercel --prod`
5. Save the deployed URL (example `https://pixora.vercel.app`).

## 2) Configure face engine mode

Use external real inference service:

1. Deploy `services/face-engine` to a Python host (Render/Railway/Fly/etc).
2. Ensure Python version is `3.10.x` or `3.11.x` (for Render, set `PYTHON_VERSION=3.10.14` if needed).
3. Set service env vars:
  - `ENGINE_MODE=real`
  - `MODEL_NAME=buffalo_l`
  - `ENGINE_AUTH_TOKEN=<strong-secret>`
4. In web production env set:
  - `FACE_ENGINE_URL=https://<face-engine-domain>`
  - `FACE_ENGINE_TOKEN=<same ENGINE_AUTH_TOKEN>`

Redeploy web after updating env vars:

- `vercel --prod`

## 3) GitHub cron worker

Workflow file is already present:

- `.github/workflows/process-worker-cron.yml`

Set repository secrets:

- `PIXORA_BASE_URL` = deployed web app URL
- `INTERNAL_WORKER_TOKEN` = same as web production env

Run once manually from GitHub Actions:

- Workflow: **Process Worker Cron**
- Action: **Run workflow**

## 4) End-to-end production validation

1. Open deployed web URL.
2. Enroll face from `/enrollment`.
3. Create group from `/groups`.
4. Upload/register a photo in group page.
5. Trigger worker once manually (or wait for cron):
   - `curl -X POST https://<web-domain>/api/v1/internal/process-next -H "Authorization: Bearer <INTERNAL_WORKER_TOKEN>"`
6. Check `/shares` for results.

## 5) If something fails

- `401 Unauthorized` on process endpoint:
  - `INTERNAL_WORKER_TOKEN` mismatch between web env and caller/GitHub secret.
- `Face engine enroll failed`:
  - `FACE_ENGINE_TOKEN` mismatch between caller and internal face-engine endpoint check.
- Worker `Object not found`:
  - photo was registered but not actually uploaded via signed URL.
- Worker keeps retrying old failed jobs:
  - clear stale failed rows in `processing_jobs` and related `photos` for test data.

## 6) Production thresholds

If local testing used aggressive values, reset in production:

- `AUTO_SHARE_THRESHOLD=0.62`
- `REVIEW_MIN_THRESHOLD=0.48`
